## 显式标注体系

这套标注在所有 skill 和代码里统一使用：

| 标注                    | 用途                       |
| ----------------------- | -------------------------- |
| `[ASSUMPTION: 内容]`    | 未经确认的推断，需要验证   |
| `[OPEN QUESTION: 问题]` | 未解决的问题，不能沉默绕过 |
| `[INFERRED]`            | 从文件或上下文推断出的内容 |
| `// FIXME: 原因 + 计划` | 临时妥协，上线前必须处理   |
| `// TODO: 内容`         | 已知待做项                 |
| `// DEVIATION: 原因`    | 有意偏离规范，说明为什么   |

## 新建 skill 的检查清单

- [ ] description 里有 "Use when..."，触发条件具体
- [ ] 执行指令全部用祈使句，没有歧义主语
- [ ] 关于用户的描述用第三人称"用户"，没有用"你"
- [ ] 说给用户的话用引号块标出，和执行指令区分开
- [ ] 必须问的信息和可以推断的信息已明确区分
- [ ] 执行逻辑和说给用户的话已分开
- [ ] 执行时序清楚：前置条件 → 信息收集 → 执行 → 关键节点 → 完成后
- [ ] 每条规则都有：保护什么 / 后果 / 偏离条件 / 偏离记录 / 自检
- [ ] 自检挂在对应规则下，每条不超过两个问题，可以直接找到答案
- [ ] 边界清楚：这个 skill 不处理什么
