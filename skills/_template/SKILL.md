## 显式标注体系

这套标注在所有 skill 和代码里统一使用：

| 标注                    | 用途                               |
| ----------------------- | ---------------------------------- |
| `[ASSUMPTION: 内容]`    | 未经确认的推断，需要验证           |
| `[OPEN QUESTION: 问题]` | 未解决的问题，不能沉默绕过         |
| `[INFERRED]`            | 从文件或上下文推断出的内容         |
| `// FIXME: 原因 + 计划` | 临时妥协，上线前必须处理           |
| `// TODO: 内容`         | 已知待做项                         |
| `// DEVIATION: 原因`    | 有意偏离规范，说明为什么在这里偏离 |

## 新建 skill 的检查清单

- [ ] description 里有 "Use when..."，触发条件具体
- [ ] 信息收集说清楚了来源和缺口处理
- [ ] 每条规则都有：保护什么 / 后果 / 偏离条件 / 偏离记录 / 自检
- [ ] 自检挂在对应规则下，不是独立章节
- [ ] 自检每条不超过两个问题，问题可以直接在代码里找到答案
- [ ] 边界清楚：这个 skill 不处理什么
- [ ] 长度合适：能说清楚就够，不追求完整
