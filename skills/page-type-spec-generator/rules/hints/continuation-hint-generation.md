# continuation-hint 生成规则

## 目标

生成 `continuation-hint.md`。

这个文件会被前页续写判定 prompt 直接读取，主要消费方是：

1. `src/module/material/ai/continuation_ai.py`
2. `src/module/material/ai/common.py`
3. `src/module/material/ai/prompts.py`

因此它必须提供稳定的结构规则，而不是泛泛的摘要性描述。

## 输入

1. 当前 type 对应的续写判定素材
2. 与内容结构、可跨页单元、终止条件相关的说明

## 总规则

1. `continuation-hint` 只负责结构归属判断，不负责 tag 取值、块组装或摘要。
2. 优先写“可跨页单元是什么、何时继续、何时终止、哪些段应忽略”。
3. 规则必须尽量可判定、可执行，避免空泛描述。

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 续写判定提示

## 可跨页单元

__PTSG_PENDING__

## 续写继续信号

__PTSG_PENDING__

## 续写终止信号

__PTSG_PENDING__

## 版式重复段 / 忽略段

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

### 可跨页单元

1. 明确什么样的结构单元可以跨页延续。
2. 如果存在固定标题、固定锚点或固定栏位，写出来。

### 续写继续信号

1. 说明哪些信号表明本页仍在延续前一页同一单元。
2. 例如：同一事项继续展开、同一结构单元未收尾、同一条目仍在延伸。

### 续写终止信号

1. 明确什么信号意味着应停止把后续段归为前页续写。
2. 优先写“新单元起始”“新节点起始”“结构锚点切换”这类强信号。

### 版式重复段 / 忽略段

1. 写哪些页眉、页脚、标题重复段、患者信息栏不参与续写判断。
2. 若这些段不能作为终止条件，也要写清。

## 写法要求

1. 多用结构词，不要只写主题概括。
2. 规则应能直接帮助模型回答“这段是否属前页续写”。
3. 若存在 partial continuation 的稳定边界，也可在继续/终止规则中写明。

## 禁止

1. 不得写标签取值规则。
2. 不得写块组装逻辑。
3. 不得把摘要模板或摘要要求写进来。

## 失败信号

1. 文件只是在描述页类型主题，没有结构规则。
2. 读完后仍无法判断“什么算继续、什么算终止”。
3. 文件主要内容与续写判定无关。
