# continuation-hint 生成规则

## 目标对象

生成 `continuation-hint.md`。

## 输入

1. 当前 type 对应的续写判定素材
2. 与内容结构、可跨页单元、终止条件相关的说明

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 续写判定提示

## 可跨页单元

__PTSG_PENDING__

## 续写继续信号

__PTSG_PENDING__

## 续写终止信号

__PTSG_PENDING__

## 版式重复段 / 忽略段

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

1. 只写续写判定规则。
2. 明确可跨页单元是什么。
3. 明确什么信号表示继续。
4. 明确什么信号表示终止。
5. 明确哪些重复段、页眉页脚或固定栏位应忽略。
6. Stage 7 必须保留 Stage 6 的四段骨架；只允许逐章节补写，不得把整份文件压成一段总说明。

## 禁止

1. 不得写标签取值规则。
2. 不得写块组装逻辑。
3. 不得把摘要模板或摘要要求写进来。
4. 不得把多章节骨架压缩成泛化待补说明。

## 失败信号

1. 文件只是在描述页类型主题，没有结构规则。
2. 读完后仍无法判断“什么算继续、什么算终止”。
3. 文件主要内容与续写判定无关。
4. Stage 6 已有章节骨架，但 Stage 7 被收缩成一两句总说明。
