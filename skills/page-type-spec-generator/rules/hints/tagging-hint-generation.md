# tagging-hint 生成规则

## 目标

生成 `tagging-hint.md`。

这个文件会被直接插入段打标 user prompt，主要消费方是：

1. `src/module/material/ai/common.py`
2. `src/module/material/ai/segment_ai.py`
3. `src/module/material/ai/prompts.py`

因此它必须写成“给 LLM 的页类型专项增量规则”，而不是给人看的解释文档。

## 输入

1. `反例`
2. `采集和转化规范`
3. 当前 type 的标签集合

## 总规则

1. `tagging-hint.md` 只写段打标阶段的页类型专项增量。
2. 系统 prompt 已说明的通用打标规则不再重复。
3. `value_hint` 已说明的单标签取值边界不再重复。
4. 优先输出“标签对 / 标签家族”的边界，而不是“一标签一句”模板。

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 段打标处理指导

## 易混淆标签边界

__PTSG_PENDING__

## 单标签规范化规则

__PTSG_PENDING__

## 版式重复段 / 特殊段

__PTSG_PENDING__

## OCR / 排版噪声

__PTSG_PENDING__

## 其他易错点

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

### 易混淆标签边界

1. 从 `反例` 中提取真正的易混淆对。
2. 同一组近邻标签，优先合并成家族级规则。
3. 推荐格式：
   - `A vs B：一句话说明区分点。`

### 单标签规范化规则

1. 仅保留当前段内即可执行的规则。
2. 适合这里的内容包括：
   1. 单位归一
   2. 枚举归一
   3. 同义词折叠
   4. 极性保留

### 版式重复段 / 特殊段

1. 只有当页类型确有特殊段处理规则时才写。
2. 例如目录页、患者信息栏、页眉页脚、结构节点行。

### OCR / 排版噪声

1. 只写该页类型特有、会影响打标的噪声。
2. 不要写所有文档都通用的 OCR 常识。

## 写法要求

1. 每条规则最好 1-2 句。
2. 直接告诉模型“如何区分 / 如何归一 / 如何忽略”，不要写长解释。
3. 优先写真正改变打标结果的规则。

## 禁止

1. 不得把同一句泛化提醒换标签名后批量罗列。
2. 不得把 `value_hint` 的内容按标签重复抄进来。
3. 不得写块组装、块摘要、页摘要策略。

## 失败信号

1. Semantic Draft 仍主要是一标签一句模板提醒。
2. 规则里大部分只是“未提及 X 不要标注 X”。
3. 文件没有真正的标签对边界，只有泛化套话。
