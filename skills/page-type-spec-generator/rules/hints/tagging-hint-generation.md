# tagging-hint 生成规则

## 目标

从 xlsx 输入生成 `tagging-hint.md`。

`tagging-hint.md` 只承载段打标阶段的补充规则。

## 输入

1. `反例`
2. `采集和转化规范`
3. 当前 type 的标签集合

## 输出

1. `tagging-hint.md`

## 总规则

1. 只写段打标阶段有用的规则。
2. 能放进结构化字段的内容，不写进 `tagging-hint.md`。
3. 不得写块组装策略。
4. 不得写块摘要策略。
5. 无素材时输出空模板。

## 生成条件

满足以下任意一条时生成实质内容，否则输出空模板：

1. 至少一个 tag 的 `反例` 有实质内容
2. 至少一个 tag 的 `采集和转化规范` 包含仅靠当前段即可执行的规则

## 应承载的内容

1. OCR / 排版噪声处理
2. 标签边界易混淆情况
3. 单页范围内有效的取值优先级与防线
4. 某类段落的特殊打标约束

## 不应承载的内容

1. 标签值定义本身
2. 结构化字段中已经表达的规则
3. 块组装顺序
4. 空块抑制
5. 摘要模板

## 生成步骤

### Step 1：判断是否有素材

动作：

1. 检查当前 type 下是否存在足以生成打标补充规则的素材

规则：

1. 若无素材，输出空模板
2. 若有素材，再继续后续步骤

### Step 2：提取标签边界问题

动作：

1. 从 `反例` 中提取易混淆边界
2. 归纳为打标时的区分规则

要求：

1. 必须写成区分原则，不得直接贴反例原文

### Step 2.1：易混淆对提取规则

1. 若反例本身是另一个 tag 的典型值，则生成易混淆对
2. 若反例只是在表达“未提及 / 无法确认 / 无值时不打”，则不生成易混淆对
3. 同一对只保留一条，不重复输出

### Step 3：提取单页范围内的补充规则

动作：

1. 从 `采集和转化规范` 中提取只影响段打标的内容

要求：

1. 只保留当前段即可执行的规则
2. 需要跨 tag、跨 block 才能执行的内容不得写入本文件

### Step 3.1：单页可执行性判断

若某条规则满足以下条件，才允许写入本文件：

1. 只需要当前段原文即可执行
2. 不依赖其他 tag 是否存在
3. 不依赖块级聚合结果

若不满足以上条件，必须移出本文件

### Step 4：组织输出

动作：

1. 按主题组织为短章节
2. 每条规则保持可执行

## 写法规则

### 必须做

1. 每条规则直接面向打标动作
2. 规则必须可执行
3. 规则应尽量短

### 禁止做

1. 不得重复 `value_hint`
2. 不得写块组装策略
3. 不得写摘要策略
4. 不得用空话占满全文

## 空模板规则

若无素材，输出：

```md
# {类型名} · 段打标处理指导

<!-- TODO: 当前设计稿未提供专门的段打标补充规则 -->
```

## 质量检查

生成后必须检查：

1. 是否只承载段打标规则
2. 是否误写组装或摘要内容
3. 若无素材，是否为空模板
4. 是否正确区分了边界冲突与无值不打

## 失败信号

以下情况说明生成失败：

1. 把 `value_hint` 重复抄进来
2. 把组装策略写进 `tagging-hint.md`
3. 把摘要模板写进 `tagging-hint.md`
4. 无素材却生成大段泛化废话
5. 需要全局视野的规则被错误写入本文件
