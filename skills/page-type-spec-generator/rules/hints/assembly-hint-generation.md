# assembly-hint 生成规则

## 目标

从 xlsx 输入生成 `assembly-hint.md`。

`assembly-hint.md` 只承载块组装阶段的执行策略。

## 输入

1. `采集和转化规范`
2. `缺项处理`
3. `冲突处理`

## 输出

1. `assembly-hint.md`

## 总规则

1. 只写块组装阶段的执行策略。
2. 不得写标签取值规则。
3. 不得写块语义定义。
4. 不得写摘要模板。
5. 无素材时输出空模板。

## 生成条件

以下任意一列有实质内容时生成实质内容，否则输出空模板：

1. `采集和转化规范`
2. `缺项处理`
3. `冲突处理`

## 应承载的内容

1. 块组装顺序
2. 归属过滤规则
3. 缺项时的块级处理
4. 冲突时的块级取舍
5. 字数控制、拆分、拼接规则
6. 空块抑制规则

## 不应承载的内容

1. 标签值定义
2. `block_types[].description` 应承担的语义范围说明
3. 块摘要输出模板
4. 页摘要输出模板

## 来源列到章节映射

1. `采集和转化规范` -> `## 采集与规范化`
2. `缺项处理` -> `## 缺项处理`
3. `冲突处理` -> `## 冲突处理`

## 生成步骤

### Step 1：判断是否有素材

动作：

1. 检查 `采集和转化规范`、`缺项处理`、`冲突处理` 中是否存在块组装级素材

规则：

1. 若无素材，输出空模板
2. 若有素材，再继续后续步骤

### Step 2：提取组装策略

动作：

1. 从 `采集和转化规范` 中提取块组装顺序、拆分、拼接、过滤规则

### Step 3：提取缺项与冲突策略

动作：

1. 从 `缺项处理` 中提取块级降级和抑制规则
2. 从 `冲突处理` 中提取块级优先级和取舍规则

### Step 4：组织输出

动作：

1. 按组装策略、缺项策略、冲突策略分节输出
2. 章节顺序必须与来源列映射保持一致

## 写法规则

### 必须做

1. 每条规则都必须直接服务块组装
2. 每条规则必须可执行
3. 规则应尽量短

### 禁止做

1. 不得重复 `value_hint`
2. 不得写块对象定义
3. 不得写摘要模板
4. 不得写与组装无关的泛化说明

## 空模板规则

若无素材，输出：

```md
# {类型名} · 块组装处理指导

<!-- TODO: 当前设计稿未提供专门的块组装规则 -->
```

## 质量检查

生成后必须检查：

1. 是否只承载块组装策略
2. 是否误写标签取值规则
3. 是否误写块语义定义
4. 若无素材，是否为空模板
5. 是否按固定章节映射组织输出

## 失败信号

以下情况说明生成失败：

1. 把 `value_hint` 内容写进 `assembly-hint.md`
2. 把 `block_types[].description` 的语义定义写进 `assembly-hint.md`
3. 把摘要模板写进 `assembly-hint.md`
4. 无素材却生成大段泛化废话
