# block-summary-hint 生成规则

## 目标对象

生成 `block-summary-hint.md`。

## 输入

1. `块摘要构造和输出组装`（主输入）
2. `采集和转化规范` 中会直接改变块摘要输出的规则
3. `缺项处理` 中会直接改变块摘要退化方式的规则
4. `冲突处理` 中会直接改变块摘要取舍方式的规则

## Stage 6：Skeleton

输出按 block 分节的骨架：

```md
# {类型名} · 块摘要生成指导

## {Block 名称}

模板：__PTSG_PENDING__
输出顺序：__PTSG_PENDING__
归一与改写：__PTSG_PENDING__
缺项退化：__PTSG_PENDING__
冲突取舍：__PTSG_PENDING__

## {下一个 Block 名称}

模板：__PTSG_PENDING__
输出顺序：__PTSG_PENDING__
归一与改写：__PTSG_PENDING__
缺项退化：__PTSG_PENDING__
冲突取舍：__PTSG_PENDING__
```

规则：

1. 每个 block 都必须有自己的小节，不得合并成一段泛化说明。
2. 若 `块摘要构造和输出组装` 已给出明确句式模板，可直接机械转移到 `模板` 槽；不得先抽象成“摘要重点”。
3. `模板` 优先保留原有槽位顺序、标点和句间结构；不要先改写成自由描述。
4. `输出顺序` 必须显式说明句间顺序、槽位顺序或固定组织顺序。
5. `归一与改写` 只写会直接改变块摘要措辞的规则。
6. `缺项退化` 与 `冲突取舍` 没有稳定来源时才用 `__PTSG_PENDING__`。
7. 每个字段都应保持短、硬、可执行；优先单行或紧凑短句，不展开背景说明。

## Stage 7：Semantic Draft

1. 只写块摘要成品如何组织，不写其他阶段规则。
2. 优先保留 `块摘要构造和输出组装` 提供的句式骨架、槽位位置和输出顺序。
3. `输出顺序` 写成最终摘要的展开顺序，不写“优先关注什么”这类泛话。
4. `采集和转化规范` 只保留会直接改变块摘要写法的规则，例如归一、合并、去重、固定顺序、固定表达；统一落到 `归一与改写`。
5. `缺项处理` 只保留块摘要的退化口径：哪些槽位可省略，缺哪些槽位时不得构造，缺项时如何降级。
6. `冲突处理` 只保留块摘要最终取舍的优先级，不展开取值过程。
7. 若某个 block 没有明确模板，但存在清晰顺序和必备信息，仍按 block 写出：
   1. 固定输出顺序
   2. 必须出现的信息
   3. 可省略的信息
   4. 不得构造的条件
8. 若设计稿已给出完整句式模板，不得把它弱化成几条“摘要应包含什么”的提示。
9. 本文件只约束块摘要正文组织，不重写全局固定定位行格式。
10. 生成后的每个 block 小节应让下游模型一眼读出：怎么写、按什么顺序写、哪些地方可改写、缺项怎么退、冲突怎么判。
11. 优先使用短句、并列短语、箭头顺序或分号连接；不要写成长段说明文。
12. Stage 7 必须保留每个 block 小节和五个槽位；一个 unit 只允许改一个 block 的一个槽位，不得整文件改写后再回填记录。

## 禁止

1. 不得把 `块摘要构造和输出组装` 弱化成泛化的“摘要重点”或“总结方向”。
2. 不得把块组装顺序、锚点选择、tag 取值规则或页摘要规则写进来。
3. 不得主要由 `block description` 复述或改写组成。
4. 不得写模型行为解释、消费方背景、系统机制说明或其他元解释噪音。
5. 不得只罗列 tag 名称或 tag 槽位，而不说明模板位置、输出顺序、缺项退化和冲突取舍。
6. 不得把与块摘要输出无直接关系的 `采集和转化规范` / `缺项处理` / `冲突处理` 原样搬进来。
7. 不得写“该 block 主要描述……”“用于帮助模型理解……”这类说明文句子。
8. 不得把 `输出顺序` 写成抽象主题词列表；必须能映射到实际句式展开。
9. 不得把 `归一与改写` 写成取值流程说明、数据血缘说明或字段来源说明。
10. 不得把多 block 多槽位骨架压缩成一段总说明。

## 失败信号

1. 明确句式模板被改写成几条泛化提醒，导致模板结构丢失。
2. 文件主体仍是 block description 的改写版。
3. 读完后仍无法知道某个 block 的块摘要应按什么顺序、什么句式组织。
4. 文件实际在讲组装、取值或锚点，而不是讲摘要输出。
5. 文件主要在罗列 tag 槽位，却没有给出模板位置、顺序、缺项退化或冲突优先级。
6. 会直接影响块摘要输出的归一、缺项或冲突规则被漏掉。
7. 文件中出现模型机制、prompt 背景、消费链路等元解释噪音。
8. `输出顺序` 读起来无法直接映射到最终摘要展开。
9. `归一与改写` 缺失，导致 Excel 中会改变措辞的规则无处安放。
10. Stage 6 已有 block / slot 骨架，但 Stage 7 把整份文件收缩成泛化待补说明。
