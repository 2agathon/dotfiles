# block-summary-hint 生成规则

## 目标

生成 `block-summary-hint.md`。

这个文件会进入块摘要 prompt，主要消费方是：

1. `src/module/material/ai/block_ai.py`
2. `src/module/material/ai/prompts.py`

它的职责不是复述 block description，而是补充“该页类型的块摘要应该如何组织和取舍”。

## 输入

1. `块摘要构造和输出组装`
2. 与块摘要直接相关的 `采集和转化规范`
3. 与块摘要直接相关的 `缺项处理`
4. 与块摘要直接相关的 `冲突处理`

## 总规则

1. 系统 prompt 已定义通用摘要格式；本文件只写 page-type-specific 的摘要重点、顺序与缺项口径。
2. 若设计稿明确给了块摘要模板，可以保留模板槽位，但要写成 LLM 可执行的摘要提示，而不是把 block description 重新抄一遍。
3. 只写摘要生成需要的规则，不写块组装算法。

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 块摘要生成指导

## 摘要重点与模板槽位

__PTSG_PENDING__

## 缺项与冲突处理

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

### 摘要重点与模板槽位

1. 若源设计稿包含明确模板，按 block 分条写出：
   1. 推荐摘要顺序
   2. 关键槽位
   3. 可省略槽位
2. 若源设计稿没有模板，但有清晰摘要重点，也按 block 写出“必须出现的信息”。
3. 模板槽位允许使用 `[标签名]` 形式，但必须服务摘要组织，而不是原样搬运整段设计稿。

### 缺项与冲突处理

1. 只写摘要层面的口径。
2. 例如：缺少次要信息时如何省略、多个候选冲突时摘要优先保留哪一类事实。

## 写法要求

1. 每个 block 的规则优先写成 1-3 条短句或短模板。
2. 摘要模板应体现“先写什么、后写什么、缺什么可跳过”。
3. 不要把 block description 整段复制过来当摘要规则。

## 禁止

1. 不得把块组装顺序、锚点选择或 tag 取值规则写进来。
2. 不得主要由 block description 复述组成。
3. 不得把页摘要规则写到这里。

## 失败信号

1. 文件主体仍是 block description 的改写版。
2. 读完后无法知道块摘要应该怎样组织。
3. 文件实际在讲组装，不是在讲摘要。
