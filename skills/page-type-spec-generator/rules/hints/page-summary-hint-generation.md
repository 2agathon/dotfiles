# page-summary-hint 生成规则

## 目标对象

生成 `page-summary-hint.md`。

## 输入

1. 当前 type 对应的页摘要素材
2. 与整页组织顺序直接相关的专项说明

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 页摘要生成指导

## 页级重点

__PTSG_PENDING__

## 组织顺序

__PTSG_PENDING__

## 特殊表达要求

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

1. 只写页级重点、页级顺序、页级表达约束。
2. 不要逐块复述内容。
3. 没有稳定素材时，宁可保留占位，也不要编造页级规则。
4. Stage 7 必须保留 Stage 6 的三段骨架；只允许逐章节补写，不得把整份文件压成一段总说明。

## 禁止

1. 不得重复通用摘要格式常识。
2. 不得把块级摘要规则搬到页级。
3. 不得把整页摘要写成块摘要列表。
4. 不得把多章节骨架压缩成泛化待补说明。

## 失败信号

1. 最终内容只是通用空话。
2. 规则仍停留在 block 级，而不是页级。
3. 文件没有给页摘要模型提供任何页类型专项增量。
4. Stage 6 已有章节骨架，但 Stage 7 被收缩成一两句总说明。
