# page-summary-hint 生成规则

## 目标

生成 `page-summary-hint.md`。

这个文件会进入页摘要 prompt，主要消费方是：

1. `src/module/material/ai/page_ai.py`
2. `src/module/material/ai/prompts.py`

它只负责页级摘要的专项要求，不重复系统 prompt 的通用摘要规范。

## 输入

1. 当前 type 对应的页摘要素材
2. 与整页组织顺序直接相关的专项说明

## 总规则

1. 只写页级重点、页级顺序、页级表达约束。
2. 不要逐块复述内容；系统 prompt 已允许参考块摘要，但页摘要必须以整页为界。
3. 没有稳定素材时，宁可保留占位，也不要编造页级规则。

## Stage 6：Skeleton

输出固定骨架：

```md
# {类型名} · 页摘要生成指导

## 页级重点

__PTSG_PENDING__

## 组织顺序

__PTSG_PENDING__

## 特殊表达要求

__PTSG_PENDING__
```

## Stage 7：Semantic Draft

### 页级重点

1. 写哪些事实最值得进入页摘要。
2. 优先写整页主轴，而不是次级细节。

### 组织顺序

1. 若该页类型天然有固定主轴，如时间顺序、诊疗顺序、结论先后，显式写出来。
2. 若没有，保持空或占位，不强行规定。

### 特殊表达要求

1. 只写该页类型特有的表达要求。
2. 例如：是否先写结论后写依据，是否必须带时间点，是否要突出主要诊断而弱化次要信息。

## 禁止

1. 不得重复系统 prompt 里的通用格式规则。
2. 不得把块级摘要规则搬到页级。
3. 不得把整页摘要写成块摘要列表。

## 失败信号

1. 最终内容只是“按时间顺序概括”这类通用空话。
2. 规则仍停留在 block 级，而不是页级。
3. 文件没有给页摘要模型提供任何页类型专项增量。
