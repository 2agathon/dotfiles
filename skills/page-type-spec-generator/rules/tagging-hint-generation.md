# tagging-hint.md 生成规则

## 定位

tagging-hint.md 的内容会以 `{tagging_hint_section}` 直接注入到段打标 user prompt 里，LLM 会直接读到它。

写法要求：**直接对 LLM 说话，告诉它对这个页类型打标时要注意什么**，不是给人看的说明文档。

---

## 生成条件

满足以下任意一条则生成，否则不生成：

1. xlsx 中至少一个 tag 的 `反例` 列有实质内容
2. xlsx 中至少一个 tag 的 `采集和转化规范` 列有涉及单 tag 规范化的内容

---

## 输出模板

```markdown
# {类型名} · 打标注意事项

{如果有易混淆对，输出第一节，否则跳过}
## 易混淆标签边界

以下标签对在原文中容易相互误判，打标前先确认语义所指：

{对每一个易混淆对输出一条，格式见下}
- **{TAG_A}** vs **{TAG_B}**：{一句话说明区分点，从反例的语义推导}

{如果有规范化规则，输出第二节，否则跳过}
## 单标签规范化规则

{对每一条有效规范化规则输出一条}
- **{TAG_X}**：{规范化要求}

## OCR / 排版噪声
<!-- TODO: 根据实际文书补充 -->

## 其他易错点
<!-- TODO: 根据实际文书补充 -->
```

## 易混淆对的提取规则

**来源**：每个 tag 的 `反例` 列。

**提取逻辑**：

反例有两种形态：

**形态 A**：反例的值是另一个 tag 的典型值
- 比如 `TAG_COMPLAINT_DURATION` 的反例是"10年前"——这是 `TAG_HISTORY_DURATION` 的典型值
- 推导：这两个 tag 构成一个易混淆对
- 区分点：从两个 tag 的 `value_hint` 差异中提炼，一句话说清楚

**形态 B**：反例描述的是一类语义边界（如"未提及"、"无法确认"）
- 这类反例说明的是"无值时不打标"，不构成易混淆对
- 处理方式：不生成易混淆对条目，这条信息已经在 `value_hint` 的"不打本标签"部分覆盖了

**去重规则**：
- TAG_A vs TAG_B 和 TAG_B vs TAG_A 只输出一条
- 同一对出现多次（来自不同 tag 的反例都指向对方）只输出一条，合并区分点描述

## 规范化规则的提取规则

**来源**：每个 tag 的 `采集和转化规范` 列。

**有效内容**（在单页范围内有意义，适合进 tagging-hint）：
- 同义词合并（如"同一项目中英文名视为同一项目"）
- 单位规范化（如"数值与单位合并呈现"）
- 枚举值归一（如"↑/↓ 统一归一为升高/降低"）
- 否认项极性保留（如"否认X"不得转写为"无X"）

注意：时间/日期格式规范（YYYY-MM-DD 等）是系统级全局约束，已在 system prompt 里硬编码，不从 xlsx 动态提取，不进 tagging-hint。

**无效内容**（跨 tag 或跨页才有意义，不进 tagging-hint，留给 block-summary-hint）：
- "同类项目去重" → 需要看到全部 tag 才能去重，单页打标时无法判断
- "冲突时以 X 为准" → 需要多个 tag 实例同时在手
- "缺少时降级为…" → 需要知道其他 tag 是否存在

**有效性判断规则**：
> 如果这条规范化规则只需要看当前段的原文就能执行，则有效；
> 如果执行它需要知道"其他 tag 打了什么"或"前面有没有类似内容"，则无效，移到 block-summary-hint。

---

## 注意事项

**写法约束**：

- 用祈使句，直接告诉 LLM 做什么，不要用"应该"、"建议"等软性表达
- 每条不超过两句话，简洁优先
- 不重复 value_hint 里已经说清楚的内容
- 没有素材的节不输出占位标题，直接跳过（除 TODO 节外）

**不应该写进 tagging-hint 的内容**：

- 标签的取值定义（已在 value_hint 里）
- 块的组装规则（属于 assembly-hint）
- 跨标签的冲突解决（属于 block-summary-hint）
- 任何需要"全局视野"才能执行的规则
