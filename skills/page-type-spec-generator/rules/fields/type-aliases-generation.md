# type aliases 生成规则

## 目标对象

生成 `types[].aliases`。

## 输入

1. 当前 type 的稳定别名来源（若有）

## Stage 6：Skeleton

1. 默认输出显式空数组 `[]`。
2. 只有在当前运行已提供稳定别名来源时，才允许机械转移为字符串数组。
3. 不得因为字段可选就静默省略。

## Stage 7：Semantic Draft

1. 若 Stage 6 已是显式空数组，直接沿用。
2. 若 Stage 6 已有稳定别名数组，直接沿用；不得润色改写。
3. 无稳定来源时，保持空数组 `[]`。

## 禁止

1. 不得从 `type_name` 拆词生成 alias。
2. 不得从 `Block` 名称、分类名称或 description 中猜 alias。
3. 不得编造历史标题、别称或缩写。

## 失败信号

1. `aliases` 被静默省略。
2. `aliases` 中出现由推断生成的别名。
3. 明明没有稳定来源，却输出了非空数组。
