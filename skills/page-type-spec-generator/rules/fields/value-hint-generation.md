# value_hint 生成规则

## 目标

从 xlsx 五列输入生成 `tags[].value_hint`。

`value_hint` 是打标阶段的直接取值指引。它必须回答：

1. 这个 tag 抓什么
2. 怎么识别
3. 不打什么
4. 输出什么格式

## 输入

1. `tag定义`
2. `触发模式`
3. `正例`
4. `反例`
5. `必须`

## 输出

1. 一个自然语言段落

## 总规则

1. 五列是合成关系，不是拼接关系。
2. 生成结果必须是自然语言段落。
3. 不得使用标签头。
4. 不得把正例或反例原样列举进结果。
5. 不得重复解释 tag 名称本身。

## 生成步骤

### Step 1：确定核心语义

动作：

1. 从 `tag定义` 提取核心抓取对象
2. 明确该 tag 的语义边界

要求：

1. 第一句必须回答“抓什么”
2. 不得只重复 tag 名称

### Step 2：提炼识别条件

动作：

1. 从 `触发模式` 和 `正例` 中抽象识别规律
2. 把规律写成自然语言条件

要求：

1. 识别条件必须是规律，不是例子列表
2. 若 `触发模式` 已足够清晰，可直接使用其抽象后的含义
3. 若 `触发模式` 很弱，则用 `正例` 补足规律

### Step 3：提炼排除条件

动作：

1. 从 `反例` 中抽象排除规律
2. 写出“不打什么”

要求：

1. 必须说明排除的语义原因
2. 不得只罗列反例文本
3. 若 `反例` 本质是“未提及 / 无法确认”，则转写为无值处理，不写成边界冲突
4. 必须先判断反例属于哪一类：
   - 边界冲突：反例其实是另一个 tag 的典型值
   - 无值不打：反例其实是在表达“原文未提供稳定可打值”

### Step 4：确定输出格式

动作：

1. 确定单值或多值
2. 确定是否需要规范化
3. 根据 `必须` 写无值处理

要求：

1. 若 `必须=是`，必须明确“原文无明确表述时不打本标签”
2. 若 `必须=否`，必须明确“无则可缺省”或等价表达

## 组织顺序

最终段落必须按以下顺序组织：

1. 是什么
2. 怎么识别
3. 不打什么
4. 输出格式

## 写法规则

### 必须做

1. 用自然语言连续成段
2. 每个语义点一句话或短句
3. 优先抽象规律，不列举原始例子
4. 优先保留对区分相邻标签有帮助的信息

### 禁止做

1. 不得写：`可打本标签的内容：`
2. 不得写：`怎么识别：`
3. 不得写：`不打本标签：`
4. 不得写：`输出：`
5. 不得把 `触发模式`、`正例`、`反例` 原样拼接进去
6. 不得把格式规则写成指令列表
7. 不得生成重复句段
8. 不得出现模板残留或拼接病句

## 特殊情况

### 1. `触发模式` 为空

动作：

1. 用 `tag定义 + 正例` 提炼识别条件

### 2. `正例/反例` 为空

动作：

1. 用现有列继续生成
2. 允许结果更短

### 3. `tag定义` 很短

动作：

1. 用 `触发模式 + 正例` 补足完整语义

### 4. 正例跨多个命中场景

动作：

1. 抽象成多种命中形态
2. 不得把多种场景混成一条模糊句子
3. 若多种场景语义差异明显，必须分句描述

## 质量检查

生成后必须检查：

1. 是否回答了“抓什么”
2. 是否回答了“怎么识别”
3. 是否回答了“不打什么”
4. 是否回答了“怎么输出”
5. 是否出现标签头
6. 是否出现原始例子列表
7. 是否出现明显拼接痕迹
8. 是否对相邻标签有足够区分度
9. 是否正确处理了反例的两种形态

## 失败信号

以下情况说明生成失败：

1. 只是在复述 tag 名称
2. 直接列举正例或反例
3. 使用标签头拼接句子
4. 输出格式未说明
5. 无值处理未说明
6. 存在重复句段、拼接残留或明显病句
7. 反例明明属于边界冲突，却被错误写成“未提及时不打”
