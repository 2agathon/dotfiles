# value_hint 生成规则

## 目标

从 xlsx 五列输入生成 `tags[].value_hint`。

`value_hint` 是段打标阶段给 LLM 的 tag 级取值契约。它直接进入运行时 prompt，被以下模块消费：

1. `src/module/material/ai/segment_ai.py`
2. `src/module/material/ai/common.py`
3. `src/module/material/ai/prompts.py` 中的段打标 prompt

它的任务不是重复系统 prompt，而是补充该 tag 专属的识别边界。

## 必须回答的问题

1. 这个 tag 抓什么
2. 主要靠什么识别
3. 哪些相近内容不要打到这个 tag
4. 值要如何保留、归一或省略

## 输入

1. `tag定义`
2. `触发模式`
3. `正例`
4. `反例`
5. `必须`

## 总规则

1. Skeleton 阶段与 Semantic Draft 阶段必须显式分开。
2. Skeleton 阶段只允许模板展开，不允许伪装成最终自然语言。
3. Semantic Draft 阶段才允许把骨架模板转成最终 `value_hint`。
4. 最终文本应是 LLM 友好的短段落，不是字段头拼接，也不是大段说明文。
5. 最终文本只保留该 tag 的专属边界；系统 prompt 已定义的通用抽取规则不再重复。

## Stage 6：Skeleton

### 输出形态

固定输出以下五行：

```text
定义：{tag定义或 __PTSG_PENDING__}
触发：{触发模式或 __PTSG_PENDING__}
正例：{正例或 __PTSG_PENDING__}
反例：{反例或 __PTSG_PENDING__}
必须：{必须列或 __PTSG_PENDING__}
```

### 规则

1. 五个槽位必须全部显式出现。
2. 某列为空时必须填 `__PTSG_PENDING__`，不得省略该行。
3. Skeleton 阶段不得把正例和反例抽象成规律句。
4. Skeleton 阶段的职责是暴露输入，不是抢先写结论。

## Stage 7：Semantic Draft

### 目标形态

最终 `value_hint` 应是 2-5 句自然语言短段落，建议按以下顺序组织：

1. 抓取对象
2. 主要识别线索
3. 排除边界
4. 取值或归一口径
5. 无值处理

### 合成步骤

#### Step 1：确定抓取对象

1. 从 `tag定义` 提炼当前 tag 的核心语义对象。
2. 第一句必须直接回答“抓什么”，不要先写空话开场。

#### Step 2：提炼识别线索

1. 从 `触发模式` 与 `正例` 中抽象识别规律。
2. 写规律，不写例子列表。
3. 仅当一个短例子能显著减少歧义时，允许保留 1 个典型表达。

#### Step 3：提炼排除边界

1. 从 `反例` 判断这是“易混淆边界”还是“无值不打”。
2. 易混淆边界写成排除句。
3. “未提及 / 无明确值 / 无法确认”一类内容写入无值处理，不要伪装成边界规则。

#### Step 4：确定输出口径

1. 明确取值是忠实抄取、轻度归一，还是结构化判读。
2. 若存在格式要求、单位要求、时间要求，只保留 tag 专属部分。
3. 根据 `必须` 写明“无明确值时不输出”或“可缺省”。

## 写法要求

1. 直接说业务边界，不要重复 tag 名称本身。
2. 尽量用具体语义词，而不是“直接对应的表达”“边界不一致的内容”这类空泛短语。
3. 允许出现 1 个典型表达，但不得出现“待补示例”。
4. 不要把 `触发模式`、`正例`、`反例` 原样粘贴到结果里。

## 禁止

1. 不得写成字段头拼接后的半结构化文本。
2. 不得用所有 tag 都能套用的通用句替代专属边界。
3. 不得用“优先识别与该标签直接对应的显式表达”作为默认主句，除非这个 tag 的核心就是显性标记值。
4. 不得输出“待补示例”“与当前标签边界不一致的内容”这类低信息量模板句。

## 失败信号

1. Skeleton 阶段缺少任一模板槽位。
2. Semantic Draft 仍停留在字段头拼接态。
3. 最终主识别句替换成任意近邻 tag 后依然基本成立。
4. 最终文本主要由空泛模板句组成，而不是当前 tag 的专属边界。
