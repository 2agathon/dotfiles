# context_hint 生成规则

## 目标

从 xlsx 输入生成 `tags[].context_hint`。

`context_hint` 的职责是约束 `tag_context` 应补充哪些最小必要语义边界。

## 输入

1. `tag定义`
2. `正例`
3. `反例`
4. 当前 tag 所属 block 的语义范围
5. 当前 tag 的 `value_hint`

## 输出

1. 一个自然语言短段落，或空值

## 总规则

1. `context_hint` 只在存在真实歧义时生成。
2. 无歧义时应留空。
3. `context_hint` 只说明需要补充的断言边界。
4. `context_hint` 不解释模型为什么这样判。

## 验收句

只看 `(tag_id, tag_value, tag_context)`，应能回答“这条实例在断言什么”；但不应从中读出“为什么这样判、位于哪里、如何归组”。

## 允许承载的内容

仅允许写以下内容：

1. 所指对象
2. 语义角色
3. 成立边界
4. 作用范围
5. 最小来源性语义

## 歧义维度

判断 `context_hint` 是否需要生成时，只允许检查以下四个维度：

1. 所指对象
2. 语义角色
3. 成立方式
4. 成立边界

## 禁止承载的内容

1. 来源位置
2. 判读依据
3. OCR 修正过程
4. 锚点归属信息
5. 组装信息
6. 大段原文证据
7. 解释性长段说明

## 生成步骤

### Step 1：判断是否需要生成

动作：

1. 判断 `tag_value` 与当前 `value_hint` 组合后是否足够表达完整断言
2. 判断该 tag 是否存在真实歧义

规则：

1. 若无歧义，输出空值
2. 若有歧义，再继续后续步骤

### Step 2：确定歧义轴

动作：

1. 判断歧义来自对象
2. 判断歧义来自角色
3. 判断歧义来自成立方式
4. 判断歧义来自边界

规则：

1. 只选择真实存在的歧义轴
2. 不得为“内容更完整”而额外补轴

### Step 2.1：歧义信号

按以下信号判断是否存在真实歧义：

| 维度 | 歧义信号 |
|---|---|
| 所指对象 | value 是通用词或通用时间，不说明指向哪个对象 |
| 语义角色 | value 本身可能对应多个语义角色 |
| 成立方式 | value 可能表示当前/既往、肯定/否定、已发生/拟定 |
| 成立边界 | value 含时间、范围、条件，但离开上下文就不完整 |

### Step 3：生成最小补充语义

动作：

1. 只写补足断言所需的最小语义
2. 优先写对象、角色、边界

规则：

1. 结果必须短
2. 结果必须可直接指导 `tag_context` 的书写
3. 不得写成长篇说明

### Step 4：留空判断

满足以下任一条件时，必须留空：

1. 四个维度都无歧义
2. 补写会重复 `value_hint` 已明确表达的内容
3. 当前判断置信度不足，补写反而会误导

## 写法规则

### 必须做

1. 用自然语言短句
2. 只写需要补足的断言边界
3. 优先指向对象、角色、边界

### 禁止做

1. 不得写：原文位于哪里
2. 不得写：因为什么判断成这个标签
3. 不得写：应如何做 OCR 修正
4. 不得写：该值如何参与锚点绑定
5. 不得写：该值如何参与 block 组装
6. 不得默认生成长篇 `context_hint`

## 特殊情况

### 1. 时间类标签

动作：

1. 仅在需要区分时间类型时补充最小语义

允许写：

1. 本次入院时间
2. 记录时间
3. 报告时间

禁止写：

1. 页面位置
2. 字段来源说明链路

### 2. 角色类标签

动作：

1. 当同一值可能对应多个语义角色时，明确当前角色

### 2.1 既往 / 当前类标签

动作：

1. 当同一值可能同时落入“当前”与“既往”时，优先补足成立方式

### 3. 节点或层级类标签

动作：

1. 当编号或标题值本身不足以区分层级时，补足层级角色

## 质量检查

生成后必须检查：

1. 是否存在真实歧义
2. 若无歧义，是否已留空
3. 若有歧义，是否只补了最小必要语义
4. 是否写入了禁止内容
5. 是否过长
6. 是否满足验收句

## 失败信号

以下情况说明生成失败：

1. 在无歧义时仍生成 `context_hint`
2. 写成解释型说明
3. 写入来源位置、判读依据或 OCR 过程
4. 写入锚点或组装信息
5. 段落明显过长
6. 明明无歧义却仍补写冗余说明
