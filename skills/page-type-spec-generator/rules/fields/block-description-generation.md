# block description 生成规则

## 目标

从 xlsx 输入生成 `block_types[].description`。

`block_types[].description` 只负责说明块的语义范围，不负责说明执行策略。

## 输入

1. `Block description`
2. `Block`
3. 当前 block 下的 tag 集合

## 输出

1. 一个自然语言段落

## 总规则

1. 必须说明块的对象是什么。
2. 必须说明块包含哪些语义维度。
3. 不得写执行策略。
4. 不得写组装顺序、字数控制、拼接策略或冲突裁决。

## 生成步骤

### Step 1：确定块对象

动作：

1. 从 `Block description` 或 `Block` 提取当前块的核心对象

要求：

1. 第一句必须回答“这个块在表达什么对象”

### Step 2：归纳语义维度

动作：

1. 从当前 block 下的 tag 集合归纳该块包含的信息维度

要求：

1. 只写语义维度
2. 不把 tag 名称简单罗列成列表句

### Step 3：去掉执行策略

动作：

1. 如果原始 `Block description` 夹带执行策略，删除这些片段

必须删除的典型内容：

1. 按时间链合并
2. 阴性症状独立成尾句
3. 超过字数时拆分
4. 缺项时如何补写
5. 冲突时如何优先取舍

## 写法规则

### 必须做

1. 用自然语言短段落
2. 先说明块对象，再说明语义维度
3. 保持块级抽象，不下沉到具体取值规则

### 禁止做

1. 不得写步骤指令
2. 不得写“应当如何组装”
3. 不得写“缺项时如何处理”
4. 不得写“冲突时以谁优先”
5. 不得把 tag 名称机械串成说明文

## 特殊情况

### 1. 原始 `Block description` 已符合要求

动作：

1. 允许轻微整理后直接使用

### 2. 原始 `Block description` 很弱

动作：

1. 用 `Block` 名称和 tag 集合补足块对象与语义维度

### 3. 原始 `Block description` 夹带大量策略

动作：

1. 只保留语义说明部分
2. 策略内容不写入 `block_types[].description`

## 质量检查

生成后必须检查：

1. 是否说明了块对象
2. 是否说明了语义维度
3. 是否夹带执行策略
4. 是否过度罗列 tag 名称

## 失败信号

以下情况说明生成失败：

1. 只是在重复 block 名称
2. 完全由 tag 名称堆砌成段落
3. 写入了执行策略
4. 写入了缺项处理或冲突处理
