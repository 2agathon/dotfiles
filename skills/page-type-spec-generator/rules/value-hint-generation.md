# value_hint 合成规则

## 定位

`value_hint` 是打标 AI 的核心取值指引。它回答的是：**这个 tag 抓什么、怎么识别、不打什么、输出什么格式**。

它会被直接注入到段打标 user prompt 里，打标 AI 会逐字读到它。

---

## 输入来源

五列共同参与合成，是合成关系，不是拼接关系：

| xlsx 列 | 贡献的语义维度 |
|---|---|
| `tag定义` | 核心语义：这个 tag 抓的是什么，语义边界在哪里 |
| `触发模式` | 识别信号：靠什么在文本里找到它（关键词、语义判断、结构位置） |
| `正例` | 命中边界的具体例子——用于推导识别条件，不直接出现在输出里 |
| `反例` | 排除边界的具体例子——用于推导排除条件，不直接出现在输出里 |
| `必须` | 无值时的处理方式（是 → 找不到则不打本标签；否 → 可缺省） |

---

## 合成步骤

### 第一步：理解语义边界

读懂 `tag定义`，明确这个 tag 的核心抓取对象是什么。

### 第二步：提炼识别条件

从 `触发模式` + `正例` 中**抽象出**识别规律：

- 不是列举正例，而是找到正例的共同特征，写成条件句
- 如果触发模式已经很清晰，直接用；如果只是关键词列表，结合正例补充语义判断条件
- 正例只作为推导原材料，不得以"例如：发热、咳嗽"这种形式出现在最终输出里

### 第三步：提炼排除条件

从 `反例` 中抽象出排除规律：

- 同样不是列举反例，而是说清楚为什么不打、排除的语义原因是什么
- 反例有两种形态：
  - **形态 A**：反例的值是另一个 tag 的典型值 → 说明边界在哪里、和哪个 tag 容易混淆
  - **形态 B**：反例描述"无值时不打标"（如"未提及"、"无法确认"）→ 这条信息进"不打本标签"节，不是排除条件

### 第四步：确定输出格式

- 单值还是多值
- 是否需要规范化（时间格式、单位、归一化等）
- 根据 `必须` 列决定无值时的表达：
  - `必须=是` → 末尾加"原文无明确表述时不打本标签"
  - `必须=否` → 末尾加"无则可缺省"或省略

---

## 合成顺序

最终输出按以下语义顺序组织，形成连贯自然的段落：

1. **是什么**：核心语义，一句话说清楚抓什么
2. **怎么识别**：识别信号和条件（从触发模式+正例推导）
3. **不打什么**：排除条件（从反例推导）
4. **输出格式**：值的形式、规范化规则、无值处理

---

## 写法约束

**应该做：**
- 用自然语言写，不用标签头（不写"取值：xxx"、"触发条件：xxx"这种格式）
- 句子之间用自然语言过渡，不用分号硬连
- 每个语义点一句话，简洁优先
- 识别条件和排除条件抽象成规律，不具体列举原文中的例子

**不应该做：**
- 把触发模式、正例、反例直接拼进去（这是最常见的干硬感来源）
- 重复 tag name 的含义（如"本标签用于标注主诉症状"这类废话开头）
- 把规范化规则写成指令列表（用自然语言融入段落里）
- 写"如有/如无"式条件句超过两个

## 示例

**xlsx 原始输入（TAG_COMPLAINT_SYMPTOM）：**
- tag定义：患者主诉症状、异常检查结果或此前已确诊疾病名称
- 触发模式：结合语义语境，患者希望本次治疗解决的问题，以及症状表现、异常检查结果或此前已确诊疾病
- 正例：发热、咳嗽、疼痛；发现空腹血糖升高1月；确诊白血病1年，入院行第4个疗程化疗
- 反例：既往反复发热
- 必须：是

**合成后的 value_hint：**
> 患者本次就诊希望解决的主要问题，包括症状表现、异常检查发现或此前已确诊的疾病名称。取原文描述，保留症状性质、发现方式等修饰信息。含"既往"修饰的历史症状不打本标签——只取本次就诊的主诉对象。原文无明确主诉表述时不打本标签。

---

**xlsx 原始输入（TAG_ABNORMAL_TYPE）：**
- tag定义：高/低
- 触发模式：↑或者↓
- 正例：↑
- 反例：未说明
- 必须：否

**合成后的 value_hint：**
> 异常指标的方向，取值为"升高"或"降低"。↑ 统一归一为"升高"，↓ 统一归一为"降低"，不保留原始符号。原文无异常方向标记时可缺省。

---

## 边界情况处理

**触发模式列为空：**
直接从 tag定义 + 正例推导识别条件，不强行补触发规则。

**正例/反例均为空：**
只用 tag定义 和触发模式合成，输出会相对简短，这是正常情况。

**tag定义 非常简短（如"高/低"）：**
从触发模式和正例里补充完整语义，不要因为原文短就输出一句话了事。

**多个正例跨越不同语义场景：**
说明这个 tag 有多种命中形态，合成时分情况描述，而不是把所有情况混在一句话里。
