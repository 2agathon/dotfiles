# context_hint 生成规则

## 定位

`context_hint` 告诉打标 AI：对这个 tag，应该在 `tag_context` 字段里补足哪些语义缺口。

它不是给人看的说明，而是直接注入到段打标 user prompt 里，打标 AI 会用它来决定 tag_context 应该写什么。

xlsx 里没有直接对应列，需要 Agent 基于已有信息推断生成。

---

## tag_context 的定位（来自 page-semantic-contract.v1.md，已冻结）

tag_context 只回答：**这条标签实例在断言什么**。

它补足的是：`(tag_id, tag_value)` 这个对子本身没有说清的断言边界。

验收标准：只看 `(tag_id, tag_value, tag_context)` 三者，不查原文，应能准确回答"这条实例在断言什么"；但不应从中读出"它为什么这样判、位于哪里、如何归组"。

---

## context_hint 应该引导补足哪些维度

tag_context 有四个可能需要补足的维度：

1. **所指对象**：该值指向谁、什么、哪一项、哪一角色
2. **语义角色**：该值在此处表示什么，而不只是字面字符串
3. **成立方式**：肯定/否定、当前/既往、已发生/拟定、要求/承诺/建议/结论
4. **成立边界**：时间点、阶段、范围、条件、段内局部范围

context_hint 不需要覆盖所有四个维度，**只写这个 tag 实际容易产生歧义的那些**。

---

## 推断步骤

### 第一步：读 value_hint

value_hint 已经说清楚了"这个 tag 抓什么"。从中判断：这个 tag 的 value 是一个自说明的值（如"升高"），还是一个离开上下文就会产生歧义的值（如"3天前"、"张三"）。

### 第二步：逐维度判断歧义

对四个维度依次判断：**如果 value 本身不能回答这个维度的问题，且不补足会导致断言不完整或产生歧义，则这个维度需要写进 context_hint**。

判断方法（举例）：

| 维度 | 歧义信号 | 示例 |
|---|---|---|
| 所指对象 | value 是一个通用词，不知道指谁/哪一项 | `TAG_COMPLAINT_DURATION`，value="3天"，不知道是哪个症状的持续时间 |
| 语义角色 | value 是编码、枚举或缩写，含义不自明 | `TAG_ABNORMAL_TYPE`，value="升高"，但需要知道是哪个指标升高 |
| 成立方式 | value 可能是当前也可能是既往，或可能是否定表达 | `TAG_COMPLAINT_SYMPTOM`，value="咳嗽"，需要知道是当前症状还是既往 |
| 成立边界 | value 有时间/范围限定，但 value 本身没有体现 | `TAG_PRESENT_ONSET_TIME`，value="3天前"，需要知道是相对哪个时间点 |

### 第三步：只写有歧义的维度

对每个确认有歧义的维度，写一句话告诉打标 AI 应该补足什么。

**没有歧义的维度不写**，不要凑字数。

如果所有维度都没有歧义（value 本身已经足够自说明），context_hint 留空，不生成该字段。

---

## 写法约束

- 用祈使句直接告诉打标 AI 应该补什么，不用"建议"、"可以"等软性表达
- 每个维度一句话，不超过两句
- 不重复 value_hint 里已经说清楚的内容
- 不描述"如何判断"，只说"补什么"

---

## 示例

**TAG_COMPLAINT_DURATION（主诉持续时间）**

value_hint 说：主诉中记录的症状持续时长，取相对时间表达。

歧义判断：
- 所指对象：有歧义——"3天"是哪个症状的持续时间，需要补
- 语义角色：无歧义——持续时间就是持续时间
- 成立方式：无歧义
- 成立边界：无歧义

生成的 context_hint：
> 说明该持续时间所属的症状对象（如"发热持续时间"、"咳嗽持续时间"）。

---

**TAG_RECORD_TIME（记录时间）**

value_hint 说：取入院日期，格式 YYYY-MM-DD 等。

歧义判断：
- 所指对象：无歧义——就是入院记录的记录时间
- 语义角色：无歧义
- 成立方式：无歧义
- 成立边界：无歧义

→ context_hint 留空，不生成该字段。

---

**TAG_HISTORY_DISEASE（既往疾病名称）**

value_hint 说：既往疾病的名称。

歧义判断：
- 所指对象：有歧义——可能有多种既往疾病，需要区分是哪一种
- 成立方式：有歧义——是现在仍有的，还是已经痊愈的，或者是否认的

生成的 context_hint：
> 说明具体是哪种既往疾病；说明当前是否仍存在（现患/既往/已愈/否认）。

---

## 什么情况下 context_hint 留空

- value 本身已经足够自说明，四个维度都没有歧义
- tag 是枚举类型且取值范围极有限（如 TAG_ABNORMAL_TYPE 只有"升高/降低"）
- 推断置信度极低，写出来反而误导

留空时不生成 context_hint 字段，不写空字符串。
