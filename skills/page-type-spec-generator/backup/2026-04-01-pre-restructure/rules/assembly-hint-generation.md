# assembly-hint.md 生成规则

## 定位

assembly-hint.md 的内容会以 `{assembly_hint_section}` 注入到块组装 user prompt 里。

它的职责是：在块组装阶段，告诉 LLM 如何处理取舍决策——采集规范、缺项降级、冲突优先级。这是这三列的**主承载位置**。

---

## 生成条件

以下任意一列有实质内容则生成实质内容，否则生成空 hint：
- `采集和转化规范`
- `缺项处理`
- `冲突处理`

---

## 内容来源与处理

| 来源列 | 进入位置 | 说明 |
|---|---|---|
| `采集和转化规范` | 第一节 | 跨 tag 的规范化规则，如同类项目去重、项目按类别归并等 |
| `缺项处理` | 第二节 | 某个必要 tag 缺失时如何降级或跳过 |
| `冲突处理` | 第三节 | 多个 tag 实例存在冲突时的取舍规则 |

---

## 输出模板

```markdown
# {类型名} · 块组装处理指导

{如果有采集规范内容}
## 采集与规范化

{来自"采集和转化规范"列}

{如果有缺项处理内容}
## 缺项处理

{来自"缺项处理"列}

{如果有冲突处理内容}
## 冲突处理

{来自"冲突处理"列}
```

---

## 空 hint 模板

```markdown
# {类型名} · 块组装处理指导

<!-- TODO: 根据实际业务补充块组装规则 -->
```
