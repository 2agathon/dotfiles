# block-summary-hint.md 生成规则

## 定位

block-summary-hint.md 的内容会以 `{block_summary_hint_section}` 注入到块摘要 user prompt 里。

它的职责是：在块摘要生成阶段，告诉 LLM 如何处理跨 tag 的语义整合问题——包括采集规范、缺项降级、冲突取舍。

这是 assembly-hint 的兜底层：assembly 阶段没处理好的问题，摘要阶段还有一次机会通过提示修正输出质量。

---

## 生成条件

以下任意一列有实质内容则生成：
- `块摘要构造和输出组装`
- `采集和转化规范`
- `缺项处理`
- `冲突处理`

无素材时生成空 hint（见空 hint 模板）。

---

## 内容来源与处理

| 来源列 | 进入位置 | 说明 |
|---|---|---|
| `块摘要构造和输出组装` | 主体，按 block 分节 | 直接对应块摘要输出逻辑，摘要模板从这里来 |
| `采集和转化规范` | 补充节 | 跨 tag 的规范化，需要全部标签在手才能执行 |
| `缺项处理` | 补充节 | 某些 tag 缺失时如何降级构造摘要 |
| `冲突处理` | 补充节 | 多个 tag 实例存在冲突时的优先级规则 |

---

## 输出模板

```markdown
# {类型名} · 块摘要生成指导

{按 block 分节，每个 block 一节}
## {Block名称}

{摘要模板，来自"块摘要构造和输出组装"列}

{如果有采集规范/缺项/冲突内容，追加以下节}
## 采集与规范化

{来自"采集和转化规范"列中跨 tag 有效的内容}

## 缺项处理

{来自"缺项处理"列}

## 冲突处理

{来自"冲突处理"列}
```
