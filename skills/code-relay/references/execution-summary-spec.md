# 执行摘要内容规格

## 定义

执行摘要是执行会话结束时输出的认识回流通信。它把执行过程中产生的认识传递给下一轮认识会话，让认识会话在生成新交接信时能吸收执行侧的发现。

## 读者

下一轮认识会话中的 AI。不是人。

## 严格三段结构

```
## 做了什么
改了哪些文件，加了什么，删了什么。
事实陈述，不解释为什么。
粒度：到文件和函数级别，不到行级别。

## 过程中发现了什么跟预期不一样的
交接信说的和实际代码不一致的地方。
文档和实现的漂移。
意外的依赖关系。
假设被证伪的地方。
没有就写"无"。

## 临时判断
这次为了推进做了哪些临时决定。
哪些需要后续确认。
哪些是"先这样，等 X 之后再看"。
没有就写"无"。
```

## 格式要求

- 放在代码块中，可直接贴给新认识会话
- 三段之外不写任何内容
- 不写背景介绍
- 不写成人类汇报稿
- 不复述思考过程
- 不写"总结"或"下一步建议"

## 第二段的判断标准

"跟预期不一样"的参照系是交接信。如果执行过程中没有加载交接信（比如是简单任务直接执行），这一段写执行过程中发现的和用户描述或文档不一致的地方。

## 第三段的判断标准

"临时判断"不是所有决定，是那些**你知道不确定但为了推进先选了一个方向**的决定。如果每个决定都有充分依据，这一段写"无"。
