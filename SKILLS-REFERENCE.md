# Skill 控制速查

这份文档帮你快速了解：每个 skill 什么时候会出现，你能对它做什么。

不用读 25 份 SKILL.md，看这里就够日常使用了。

---

## 治理层

大部分 skill 不只做事，还约束 AI 做事时的行为。每个 skill 通过 `metadata.governance` 声明自己在任务生命周期中的角色（hook 类型）。你不需要记这些，但它解释了为什么 AI 会在特定时刻主动停下来、要求确认、或产出结构化报告。

协议细节见 `specs/governance-protocol.md`。

---

## 始终生效

这些 skill 不用触发，AI 在对应场景自动遵守。你可以在运行中干预：


| Skill       | 什么时候生效 | 你能做什么                                                    |
| ----------- | ------ | -------------------------------------------------------- |
| identity    | 始终     | 纠正 `[ASSUMPTION]`；回答 `[OPEN QUESTION]`；确认或否认 `[PREMISE]` |
| code-principles | 写代码时   | 质疑或接受 `// DEVIATION:`；偏离时 AI 必须说明原因                      |
| code-modify | 改现有代码时 | 限定改动范围；范围膨胀时选「一起改 / 只改 A / 先讨论」；确认或拒绝高风险改动声明             |
| bail-out    | 迭代修改中  | 说「别改了/回退/重来」直接触发止损；说「继续试」接受风险（AI 仍持续监测）                  |

## AI 可能主动提议

AI 识别到信号后会建议启用这些 skill。你可以接受、拒绝、或重定向。


| Skill                | AI 什么时候会提议       | 你的选项                                    |
| -------------------- | ---------------- | --------------------------------------- |
| assumption-audit     | 结论强于证据、词比定义跑得快   | 接受审计 / 纠正分类 / 不需要                       |
| conversation-to-spec | 对话接近承诺但还没落地      | 确认冻结 / 选模式（行动/设计/实验/词汇）/ 不冻结            |
| role-lens            | 对象被单一透镜主导、结论过于完整 | 接受挑战透镜 / 转 assumption-audit / 不需要       |
| drift-detector       | 边界或术语发生无声变更      | 承认 / 撤回 / 拆分 / 保留张力 / 不处理               |
| vibe-plan            | 新功能或边界不清晰的任务     | 说「不用 plan / 直接做」跳过；说「继续 {功能名}」恢复已有 plan |

## 你说了才触发

以下 skill 只在你明确要求时启用。

### 工程执行


| Skill         | 触发词                         | 关键决策点                      |
| ------------- | --------------------------- | -------------------------- |
| file-creation | 新建 / 创建 / 加一个；audit / 审目录   | 从候选路径中选；重叠时选合并还是新建         |
| code-test     | 写测试 / 加测试 / 测一下             | 无框架时选框架                    |
| code-review   | review / 审一下 / 提交前检查        | 接受 / 要求改动；也可由 git-commit 前置触发 |
| code-relay    | 认识会话 / 扫描模块 / 生成交接信；收工 / 打包 | 选模式（认识/加载/收工）；加载后 AI 等你下指令 |
| git-commit    | commit / 提一下                | WIP 还是正式；是否推送；main 分支需确认   |
